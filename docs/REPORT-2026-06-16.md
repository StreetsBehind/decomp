## 0. The one-paragraph version

We set out to find the best way to decompose a thin plan into atomic build work, and the program
converged on a sharper, more useful claim. **The edges that matter are not the ones the build catches
for free — they are the silent, expensive-to-recover obligations (authz, tenancy, mass-assignment,
audit, validation), and *every* model tier misses them by default.** Paying for a frontier model barely
helps; what helps is a harness that *supplies the obligations*. We built that harness — a **frozen
skeleton + retry over isolated cheap builds** — and showed it equals a bare frontier model (opus) on
epic cohesion at **$0**, and *beats* it once an epic grows past ~9 surfaces, because the frontier model
silently erodes its own cross-cutting uniformity as the epic gets bigger while the skeleton-driven cheap
harness holds flat. The newest (uncommitted) result dissects *why the skeleton works*: it solves two
independent problems at once, and each half of the skeleton fixes exactly one of them.

## 1. What the program is actually asking

The question started as: **"what is the best method for decomposing a thin plan into atomic
work-packet beads?"** and was sharpened twice.

1. **The generative reframe.** Decomposition isn't transcribing an already-decomposed lock — that's
   trivial and a pure-code expander "wins" it. The real problem is *generative*: turning "users can log
   in via SSO" into the dozens of atomic packets nobody enumerated (callback route, session store, token
   refresh, CSRF, error states…). The operating position:
   > **decomposition = deterministic scaffold + generative fill + deterministic verification.**
   > Determinism doesn't vanish; it relocates to the harness and the checkers.

2. **The cheap-vs-frontier pivot** (`PROPOSAL.md`, the current north star):
   > "Can good infrastructure and harness design close the affordable-vs-best gap for real software
   > work?" — i.e. can a **cheap/local model + the right harness (the *contender*)** match a
   > **frontier model run bare (the *bar*)**, at materially lower cost?

   The proposal splits software-building into five stages — **Plan → Decompose → Build → Test →
   Validate** — each measured the same way: (a) measure the frontier-bare-vs-cheap-bare gap, (b) draw
   the cost-vs-quality curve, (c) rank the harness levers, (d) verdict. **We started with Stage 3
   (Build)** because it's highest-relevance and can use executable tests as a free oracle (no LLM judge).

## 2. The conceptual spine: completeness is cost-of-omission, not coverage

Before any model run, two survey/reframe efforts fixed what we should even be measuring.

### 2.1 No one has a completeness guarantee (`PRIOR-ART-COMPLETENESS.md`)

A six-angle cross-discipline survey (requirements engineering, formal methods, safety engineering,
testing/inspection, process engineering, ecology) reached a hard conclusion:

> **"No discipline has a completeness _guarantee_ for decomposition."** The best prior art delivers
> either (1) a measurable lower bound on what's still missing, or (2) a typed floor that raises the
> floor — never a proof.

- **Conservation of difficulty is real** (≈ Tesler's Law): residual difficulty *relocates* (to
  catalog-completeness, to the all-generators-missed class) rather than vanishing.
- **Convergence/saturation (capture-recapture) is the only family that yields a *number*** — an estimate
  of the residual, not a binary or a gain. **Caveat that matters for everything downstream:**
  capture-recapture needs ~4–5 *independent* generators and is **structurally blind to anything all
  generators miss.** For correlated LLM extractors that's the worst case — it biases toward false
  completeness.

Imported techniques: Leveson's RSM completeness criteria (typed floor), Arora/Sabetzadeh/Briand
domain-model **orphan detection** (near-linear sensitivity r>0.96), mutation testing, STRIDE-per-element,
dialectical/devil's-advocate inquiry, and Lincoln-Petersen mark-recapture (Eick 1992 / STADS 2021).

### 2.2 The build-tolerant reframe (`BUILD-TOLERANT-REFRAME.md`)

> "We do **not** need to discover every edge before building… the build process is itself an
> edge-discovery engine."

This produces the **2×2 that organizes the whole program**:

| | **Cheap recovery** | **Expensive recovery** |
|---|---|---|
| **Self-revealing** (compiler/test catches it) | most dataflow wiring → *let the build find it* | architecture/data-model/ordering → discover upfront |
| **Silent** (ships clean, detonates later) | low priority | **security / privacy / tenancy / consistency / idempotency / compliance → the LETHAL QUADRANT** |

The thesis: **stop spending discovery effort on the cheap quadrant the build recovers for free; spend it
all on the silent+expensive quadrant.** This is what reframed the target from "all edges" to
"obligations."

## 3. The lethal quadrant is real — the two kill-tests (`KILL-TESTS.md`)

The reframe would be academic if the lethal quadrant didn't separate on real artifacts. Two independent
kill-tests, run 2026-06-11, confirmed it.

- **KT#1 — cost-weighted re-score of the hearth oracle.** Tagged all **162 canonical hearth edges** on
  the cost-of-omission 2×2. Result: **37% lethal (60/162)** — silent+expensive, typed exactly as
  predicted (authz / authn / tenancy / audit / seat-limit / SSO-validation), and **separable**. The
  conservative high-confidence-only floor is still **14.2% (23 edges)**. Crucially, **58% of a uniform
  edge-recall score is cheap-quadrant edges the build recovers for free** — so a method can post a
  healthy uniform recall while missing most of the lethal quadrant. A blind independent re-classification
  by a free-model panel called **58 edges lethal vs the authored 60 (within 3%)**, 79% bit-agreement —
  the typing is robust, not an authoring artifact.
- **KT#2 — build history as ground truth.** Mined ~1,360 app beads (autonomous-build + ForgeFlow /
  PickleMatch / ExchangeManager). Found **~11 concrete shipped-then-detonated lethal-quadrant misses**
  (authz/RLS, tenancy, consistency, race, data-loss, prod-blinding observability) — and **every single
  one passed the automated gate**, caught only later by smoke/audit/user-report. The data
  *structurally under-records* silent misses (the gate manufactures evidence for self-revealing misses
  and is blind to silent ones), so **~11 is a floor, not an estimate** — "the most undercounted number
  in the corpus."

**What this forced — the §10 ρ re-spec (now *substantiated*, no longer pending):** report recall **per
quadrant**, never an aggregate median; gate any deferral claim on a **`lethalEdgeRecall` veto**. Left as
banked debt: Step-2/3 snapshots were never persisted, so the literal "do two methods swap rank"
re-score couldn't run — **persist snapshots going forward.**

## 4. The proof staircase — what survived and what was killed (`STAIRCASE-RESULTS.md`)

A four-step ladder to test whether "edges = a deterministic join over typed produces/consumes" is a
sound mechanism before building on it.

| Step | Test | Result | Verdict |
|---|---|---|---|
| **0** | Ruler accuracy (mutation-test the recall + join scorers, 36 assertions) | rulers are transitive & exact-match-correct | ✅ **PASS** |
| **1** | Join-ceiling on sso-greenfield (hand-authored produces/consumes) | **88.2% ceiling (15/17), 100% join precision**; the 2 misses are intrinsic ordering/containment | ✅ **GO** |
| **2** | Live extraction A/B: model-extracted interfaces → join, vs a `depends_on` baseline | extraction→join **LOSES by −17.6 pts** (haiku 16.5 vs 34.1; sonnet 34.1 vs 51.8), **replicated** across both annotators | 🔴 **NEGATIVE — edge-join archived** |
| **3** | Headroom pre-flight on hearth (blind seam recall) | **seam recall 22.2% ± 0.9%** — materially >0 and ≪100%, so the fixture can move the needle | ✅ **GO** |

The Step-2 kill is load-bearing: "**Model strength isn't the fix** — the bottleneck is structural
(vocabulary alignment + over-wiring)." Sonnet extracted genuinely better than haiku and *still* lost.
This archived the edge-join mechanism and, in hindsight, was "an argument about the cheap quadrant" the
build handles anyway — consistent with the build-tolerant reframe.

## 5. The archetype-premise experiment (`ARCHETYPE-PREMISE-EXPERIMENT.md`)

Tested whether priming a decomposer with authored **archetypes** (reusable feature patterns) lifts
recall of surfaces+obligations without anchoring. A clean 3-arm design — **A0 blind / A1
placebo (token-matched generic advice) / A2 primed (archetype injection)** — on sonnet + haiku, K≥3,
with opus reserved as the union-builder/adversarial verifier.

- **The authoring finding:** **all six authored archetypes smuggled edges and had to be repaired.** Each
  archetype, run through contrast-pair validation, was caught hardcoding a `from→to` edge (e.g.
  oidc-sso "persist Identity to a user record" presupposes a sibling store) that violates the "no edge at
  intake" rule — demoted to a typed *obligation*. **Empirical proof that authored archetypes are
  dangerous without a curation gate**, and that a literal regex lint is too weak (semantic smuggling).
- **Verdict: partially validated — necessary-not-sufficient.** Archetypes are a **floor-raiser, not the
  solver** — one coverage cue among several (alongside orphan-detection, STRIDE-style enumeration). The
  positive result only licenses "hand-tuned priming lifts covered recall without anchoring on this
  fixture"; it can never promote an archetype to canon (that needs recurrence across ≥2 provenance
  classes).
- Produced the **hearth fixture**: an 8-feature B2B workspace plan, 84-edge manifest (47 intra + **37
  seam**), where seams are the load-bearing lethal-quadrant test set.

## 6. The reconciliation — two blind forks became one spine (`RECONCILIATION.md`)

A pre-registered v2 program (granularity × deferral × build-economics) and the archetype/obligations
line were run blind and **converged on the same spine: build-as-discovery** — "don't enumerate upfront
what the build recovers cheaply." The keystone finding that unified them:

> **The repair-premium ρ is quadrant-censored, not scalar.** Self-revealing edges have ρ≈0 (the build
> catches them); silent-expensive edges have ρ≈∞. v2's "if median ρ < 3, defer" rule **averages over the
> quadrant where deferral is fatal** — and worse, the apparatus is censored (a silent omission produces
> no repair event, so it never enters the ρ distribution at all).

This produced **7 revisions to RESEARCH-PROGRAM §10** (ρ per-quadrant; partition the endpoint + lethal
recall veto; retarget the risk score τ at the lethal quadrant; restore the obligations layer;
multi-feature corpus with intra/seam tags; saturation-residual over ≥4 decorrelated generators instead
of a coverage ratio; treat the oracle as incomplete-by-construction).

## 7. The quantitative heart — the build-gap study (M0 → M-coh-3)

This is the main body of new measurement. All grading is by a **hidden executable oracle** (no LLM
judge), cheap tier = the free **jnoccio** gateway (a high-variance free-model *mixture*), frontier =
sonnet and opus. Costs are real method-side dollars.

### 7.1 M0 — the bare baseline: obligation-blindness is (mostly) tier-independent

Bare model, happy-path spec only, hidden two-bucket oracle (happy-path vs the silently-missed
*obligation*):

- **Both tiers pass the happy path (~100%) and both skip the non-idiomatic obligations (0%)** — authz,
  mass-assignment, body-validation are floored at 0% for the cheap pool *and* sonnet. The only obligation
  added unprompted is the **idiomatic tenancy filter** (both tiers do it).
- **Opus partially breaks tier-independence:** `update-profile` authz+mass-assign goes **0% (sonnet) →
  100% (opus)** unprompted — but `post-comment` body-validation stays **0% even on opus**. So the bar is
  specifically **opus**, and it still has holes.
- **The obligation floor is both tier-independent AND size-independent** (0% at atomic *and* at the
  size-3 monolith for the non-idiomatic obligations). **Methodological consequence:** a bare-model size
  sweep over obligations is flat-at-0 and uninformative — you need a metric that isn't floored. That
  drove the cohesion turn.

> Headline: **"This is not primarily a cheap-vs-frontier gap — it's a spec/harness gap that hits every
> tier."** Good for the thesis: if frontier isn't worth paying for here, cheap + the right harness can
> match it.

### 7.2 M-coh — the cohesion turn: a metric that isn't floored

New instrument (`DESIGN.md`): the **workspace epic** (5 connected multi-tenant surfaces with a deliberate
addMember→postComment membership seam) and a **3-bucket oracle**, scored at *epic* scope:

- **Wire** — independently-built modules import and call each other.
- **X-CUT** (cross-cutting uniformity) — the fraction of applicable surfaces that actually enforce each
  obligation (tenancy/authz/mass-assign). *Not floored* — it moves continuously with size and harness.
- **Integ** — end-to-end flows that span ≥2 modules (the seam + cross-org isolation).

The oracle is mutation-tested (`scale-oracle-selftest.mjs`, in `npm run selftest`).

### 7.3 The ladder at one size (workspace epic, D1/N5)

| build config | $/epic | wire | happy | X-CUT | integ | epic✓ | reading |
|---|---|---|---|---|---|---|---|
| cheap-isolated (naive `/build-batch`) | $0 | 96% | 90% | 31% | 27% | 0/5 | per-task ok, no cohesion |
| cheap-isolated-retry | $0 | 100% | 95% | 37% | 27% | 0/5 | reliability ≠ cohesion |
| cheap-skeleton | $0 | 92% | 95% | 91% | 60% | 3/5 | cohesion solved, reliability caps |
| **cheap-skeleton-retry** | **$0** | 100% | 100% | **100%** | **100%** | **5/5** | **= the opus bar, free** |
| sonnet-whole bare | $0.03 | 100% | 100% | 29% | 67% | 0/3 | frontier ≠ enough, bare |
| **opus-whole bare** (THE BAR) | $0.25 | 100% | 100% | 100% | 100% | 3/3 | the only bare win |

Three findings:
1. **The cohesion gap is real and per-task quality is blind to it.** Happy-path is 90–100% *everywhere*,
   yet X-CUT/integ collapse for everything except opus. "**The thing you pay a frontier model for here is
   not writing the function — it's enforcing tenancy/authz uniformly across the whole epic.**"
2. **"More context" is not the fix.** sonnet-whole-bare *sees the entire epic* and still scores 29%
   X-CUT. You can't reach the bar by handing the cheap model more context — the harness has to supply the
   obligations.
3. **The two levers are orthogonal and both necessary** (M-coh-1.5): skeleton alone solves cohesion but
   reliability caps it at 3/5; retry alone solves reliability (wire 100%) but leaves cohesion untouched
   (X-CUT 37%, 0/5); **skeleton + retry = 5/5 perfect cohesive epics at $0**, matching opus-bare.

### 7.4 M-coh-3 — the size × harness surface (the money chart)

Scaled the epic up a parametric ladder `scale-d{1..4}` (D lexically-distinct domains, 4D+1 surfaces,
same hidden obligations each):

| size (D / N) | bar X-CUT | con X-CUT | bar EPIC✓ | con EPIC✓ | bar $/epic | con $/epic |
|---|---|---|---|---|---|---|
| D1 / N5  | 100% | 97% | 100% | 80% | $0.28 | $0 |
| D2 / N9  | 94%  | **97%** | 67% | 60% | $0.36 | $0 |
| D3 / N13 | 78%  | **93%** | 33% | 20% | $0.39 | $0 |
| D4 / N17 | 80%  | **95%** | 0%  | 0%  | $0.43 | $0 |

- **The bar degrades with size.** opus-whole's X-CUT slides **100→94→78→80%** and its cohesive-epic rate
  **collapses 100→67→33→0%** as the epic grows 5→17 surfaces — while wire/happy/integ stay 100%. It
  doesn't break by drift or by dropping a feature; it **silently drops a hidden cross-cutting guard on
  some surface** as it has to hold more of them in one context. The casualty is named and monotone: the
  first and most consistent is **`authz@add*Member`** (absent in 0 domains at D1 → all 4 domains at D4).
  And it gets *more expensive* as it gets worse ($0.28→$0.43).
- **The contender is size-flat.** cheap-skeleton-retry holds **X-CUT ≈ 93–97% and integ ≈ 92–100% flat
  across the whole ladder at $0**, because the frozen skeleton re-states every obligation to every
  isolated builder regardless of epic size, and each builder sees only one small surface. **It crosses
  *above* opus on the lethal-quadrant metric at N≈9 and pulls away** (+15 pts by N≈13–17).
- **Controls isolate the skeleton as the single active ingredient:** every non-skeleton cheap shape
  (isolated-retry, cheap-whole) is flat-low (X-CUT 37–48%) with 0 cohesive epics at every size. The only
  two conditions that *ever* produce a cohesive epic are opus-whole (small N) and cheap-skeleton-retry
  (all N).
- **The all-or-nothing EPIC✓ falls to 0 for both by N≈17 — but for different reasons.** The bar's zero is
  **systematic** (a real authz hole); the contender's is **statistical** (X-CUT 95% over ~34 checks means
  ~1 stray miss kills the *perfect*-run requirement). The contender's residue is a **reliability**
  problem, closable by a per-surface obligation checker (the M-coh-2 lever) — not a cohesion problem.

**The deliverable for the decompose stage:**
- **`harness*` = frozen-skeleton + isolated cheap chunks + retry**, ~1 surface/chunk.
- **`s*` (size knob): do not pour more than ≈ one workspace-sized cluster (~5 surfaces / 2 cross-cutting
  obligations) into a single bare frontier context.** That's opus-whole's cohesion ceiling; above it,
  decompose into isolated chunks under a frozen skeleton (the skeleton flattens the cohesion-risk side of
  the U-curve, so finer decomposition costs no seam tax).

## 8. NEW & uncommitted — M-coh-2 skeleton ablation: *why* the skeleton works

The committed docs flag the skeleton ablation as "not yet run." It **has been run** (uncommitted run
JSON `studies/build-gap/runs/mcoh2-{none,shape,obl,full}-k5.json`, workspace epic, cheap tier, K=5, $0)
and is **not written up anywhere yet.** Re-derived for this report, it's a clean **double dissociation**:

| skeleton variant | wire | happy | X-CUT | integ | EPIC✓ | what it pins |
|---|---|---|---|---|---|---|
| **none** (retry only, no skeleton) | 100% | 90% | 40% | 27% | 0/5 | nothing |
| **shape-only** (data shapes, no obligations) | 100% | 100% | 34% | **67%** | 0/5 | the seam / interface drift |
| **obligations-only** (rules, no shapes) | 100% | 90% | **86%** | 40% | 0/5 | the cross-cutting uniformity |
| **full** (both) | 100% | 100% | **97%** | **100%** | **80%** | both → a cohesive epic |

The per-test breakdown makes the dissociation unambiguous:
- **shape-only** lifts the membership seam to **SEAM+ 100% / SEAM- 100%** (both sides agree on the
  representation) but leaves cross-org **ISO at 0%** and the authz/tenancy crosscuts mostly at 0% — it
  fixes *drift*, not *obligations*.
- **obligations-only** lifts tenancy/authz crosscuts to **~100%** but drops **SEAM+ to 40%** (the two
  sides of the seam disagree on shape because nothing pinned it) — it fixes *obligations*, not *drift*.

**This empirically confirms the M-coh-1 hypothesis** that the skeleton solves two distinct problems
simultaneously — **(a) interface drift** (the shape clause buys integration) and **(b) cross-cutting
fragmentation** (the obligations clause buys X-CUT) — and that **neither half alone yields a cohesive
epic (both 0/5); only the union does (80%).** It's the cleanest single result in the program for
attributing the mechanism. *(This should be promoted into `RESULTS.md` and committed.)*

## 9. The infrastructure that makes it cheap

- **The free-model supply (jnoccio gateway).** `makeGatewayInvoke` calls an OpenAI-compatible gateway
  on `127.0.0.1:4317` that adaptively routes each call across many free upstreams and returns the
  *resolved* model + request id (recorded for reproducibility). Live smoke: 10/10 valid snapshots across
  9 distinct upstreams, 0 truncations, **$0**. Known hazards, all handled: per-model output cap (set
  `max_tokens=16384`), JSON-disobedience, timeouts → **retry-on-invalid re-routes past a weak draw**.
- **The retry lever is load-bearing.** Free models are flaky on real decomposition (~50% of first hearth
  draws come back empty/aborted); the structural retry (gate = "module imports+exports the surface fn,"
  no behaviour/obligation leakage) recovers them. It closes the *reliability* gap, which is orthogonal to
  cohesion.
- **`gen-skeleton.mjs`** (newest apparatus) synthesizes a frozen skeleton from the brief + data preamble
  + surface specs via `--transport gateway|sonnet|opus`, writing the skeleton + a cost-metadata sidecar
  — the instrument for the open **provenance** question (§10).

## 10. What's settled vs. open

**Settled (evidence in hand):**
- The lethal quadrant (silent + expensive obligations) is real, separable, ~37% of edges, and the part
  that ships and detonates. Uniform recall is censored by the cheap quadrant.
- Obligation-blindness is a spec/harness gap, not a model-tier gap (opus recovers *some* but not all).
- For a cohesive epic, **frozen-skeleton + retry over isolated cheap builds = bare opus at $0**, and
  **beats it above N≈9 surfaces** because the frontier model's cross-cutting uniformity erodes with size
  while the harness holds flat.
- The skeleton works by fixing two independent failure modes; each half fixes exactly one (Part VIII).
- Edge-discovery via model-extracted local interfaces (the "edge-join") **loses** to a trivial baseline —
  archived.

**Open (apparatus exists, runs pending):**
- **M-coh-2.5 — skeleton provenance** (`gen-skeleton.mjs` ready): is the win "all free," or "one
  amortizable frontier skeleton call + N free builds"? Both are wins, but they're *different claims*.
  This is the crux caveat on the entire cohesion result.
- **M-coh-2 checker lever** — a per-surface obligation checker + repair to close the contender's
  *statistical* EPIC✓ residue at scale (the X-CUT≈95% → not-quite-perfect-run gap). Doubly motivated by
  M-coh-3.
- **The "C" campaign** — the hearth 3-arm obligation-priming sweep — is **paused on two human decisions:**
  (1) control the method model (the free pool is an uncontrolled high-variance mixture; `@gateway` ≈
  `@random-free-model`), and (2) calibrate the judge (E0.4) before trusting lethal-recall numbers.
- **The other four stages** of the proposal (Plan, Test, Validate) are unmeasured; only Build has a
  verdict.
- The two side branches (`mcoh-2-provenance`, `study/oneshot-capacity`) are frozen ancestors of the
  current branch — their findings are already folded into the above; no orphaned results.

## 11. Where the MoE / ensemble question slots in

The open question that prompted this report — *would decomposing with a Mixture-of-Experts catch more
edges?* — lands precisely on the program's one numeric lens. Three findings constrain the answer in
advance:

1. **Capture-recapture is the only thing that estimates the residual, and it needs ~4–5 *independent*
   generators and is blind to shared blind spots** (§2.1). An ensemble of correlated decomposers is the
   worst case — it inflates apparent completeness without catching the lethal misses.
2. **The misses that matter are systematic, not random** (§7.4: `authz@add*Member` erodes monotonically;
   §7.1: every tier floors the same non-idiomatic obligations). Sampling-diversity (N runs of one prompt)
   shares those blind spots; only **lens-diversity** (an authz lens, a tenancy lens, a lifecycle lens)
   has uncorrelated error.
3. **We already have the converging filter and the ground truth:** the recurrence rule (≥2 provenance
   classes), the cost-weighted lethal-recall scorer, and KT#1's tagged 60 lethal edges. So the MoE claim
   is *testable*, not philosophical: measure whether a lens-diverse ensemble's union recovers lethal
   edges that **every** single pass misses, and at what precision cost.

The next experiment (proposed M-coh-4 / lens-ensemble probe) is to run that residual measurement on the
hearth lethal set before scaling the expert count.
